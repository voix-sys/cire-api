# CIRE X Growth Execution v1 (Policy-Safe)

## Objective
Drive qualified developer/agent traffic to CIRE (`/docs` -> `/v1/register` -> first API call) using X.

## Content Mix (weekly)
- 60% curation (3 posts)
- 30% CIRE use-case/product (1-2 posts)
- 10% insight/ops note (0-1 post)

## Safety Rules (hard)
1. No bulk DM promotion.
2. No repeated link-only posts.
3. No unverifiable numeric claims (e.g., "4x") unless benchmark source is attached.
4. Same destination link spacing >= 72h (unless major update).
5. Hashtags <= 3, only relevant.

## Weekly Cadence (KST)
- Posting window: 20:00-22:00 KST
- Frequency: 1 post/day (Mon-Fri)
- Replies: 10-20/day (value-first, no promo dump)

## CTA AB Test
- Variant A: "Try CIRE"
- Variant B: "See the difference"

Track weekly:
- profile clicks
- docs opens
- register count
- first-call count
